  <action>
1. Write `pool-app/prisma/schema.prisma` with the exact schema from RESEARCH.md:
   - generator client with provider "prisma-client-js"
   - datasource db with provider "postgresql" and url from env("DATABASE_URL")
   - enum JobStatus { DRAFT, SUBMITTED }
   - model Job with fields: id (cuid), name (String?), jobNumber (String? @map("job_number")), status (JobStatus @default(DRAFT)), submittedBy (String? @map("submitted_by")), submittedAt (DateTime? @map("submitted_at") @db.Timestamptz(6)), photos (Json @default("[]")), formData (Json? @map("form_data")), createdAt (DateTime @default(now()) @map("created_at") @db.Timestamptz(6)), updatedAt (DateTime @updatedAt @map("updated_at") @db.Timestamptz(6))
   - @@map("jobs") and @@index([status, createdAt(sort: Desc)])

2. Create `pool-app/src/lib/db.ts` with the Prisma singleton pattern:
   - Import PrismaClient from "@prisma/client"
   - Store on globalThis in development to prevent connection exhaustion on hot reload
   - Export as `db`
   - Log queries in development: log: process.env.NODE_ENV === "development" ? ["query"] : []

3. Create `pool-app/src/types/index.ts`:
   - Export type JobStatus = "DRAFT" | "SUBMITTED"
   - Export type CreateJobInput = { name?: string; jobNumber?: string }

4. Create `pool-app/src/lib/validations/job.ts` with Zod schema:
   - Import z from "zod"
   - createJobSchema: z.object({ name: z.string().min(1).max(200).optional(), jobNumber: z.string().max(50).optional() }).refine(data => data.name || data.jobNumber, { message: "Either job name or job number is required" })

5. Replace `pool-app/src/app/globals.css` content with:
   ```css
   @import "tailwindcss";

   @theme {
     --spacing-touch: 48px;
     --spacing-touch-lg: 56px;
     --spacing-touch-gap: 8px;
     --font-size-input: 16px;
     --font-size-body: 18px;
     --font-size-heading: 24px;
     --color-text-primary: oklch(0.141 0.005 285.82);
     --color-text-secondary: oklch(0.274 0.006 286.03);
     --color-bg-primary: oklch(1 0 0);
     --color-bg-secondary: oklch(0.985 0.002 247.84);
     --color-border: oklch(0.871 0.006 286.29);
   }

   input, select, textarea {
     font-size: max(16px, 1em);
   }
   ```

6. Replace `pool-app/src/app/layout.tsx` with the iPad-optimized layout from RESEARCH.md:
   - Import Metadata and Viewport from "next"
   - Import Toaster from "sonner"
   - Import "./globals.css"
   - metadata: title "Pool Field Forms", description, appleWebApp { capable: true, statusBarStyle: "default", title: "Pool Forms" }
   - viewport: width "device-width", initialScale 1, maximumScale 5, viewportFit "cover", themeColor "#ffffff"
   - Body: className="bg-white text-zinc-950 antialiased", children + Toaster position="top-center" with style fontSize "16px" padding "16px"
   - Add apple-touch-icon link in head

7. Create `pool-app/src/app/manifest.ts`:
   - Export default function returning MetadataRoute.Manifest
   - name: "Pool Field Forms", short_name: "Pool Forms", start_url: "/", display: "standalone", background_color: "#ffffff", theme_color: "#ffffff"
   - Icons: 192x192 and 512x512 PNG placeholders

8. Replace `pool-app/src/app/page.tsx` with a minimal placeholder:
   - Server component that just renders a heading "Pool Field Forms" and paragraph "Job list coming in Plan 02"
   - This proves the layout and CSS tokens render correctly

9. Generate Prisma client: `npx prisma generate`
   Note: `npx prisma db push` requires DATABASE_URL to be set. If Neon is not yet provisioned, the generate step validates the schema. The user will set DATABASE_URL before Plan 02.

IMPORTANT per CONTEXT.md locked decisions:
- 48px minimum touch targets, 56px for primary actions (--spacing-touch and --spacing-touch-lg)
- Dark text on light background, zinc/neutral palette (oklch values above)
- 16px base minimum on all inputs (the input/select/textarea rule)
- 18px for body text (--font-size-body)
