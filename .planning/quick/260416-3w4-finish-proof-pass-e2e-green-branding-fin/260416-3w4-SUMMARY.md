## Task 1: Playwright config — port 3001 isolation

**Changed:** `pool-app/playwright.config.ts`

- `baseURL`: `http://localhost:3000` → `http://localhost:3001`
- `command`: `npm run dev` → `next dev -p 3001`
- `url`: `http://localhost:3000` → `http://localhost:3001`
- `reuseExistingServer`: `!process.env.CI` → `false` (always fresh)
- Added `timeout: 120000` for Next.js warm-up

Root cause of the original failure: `reuseExistingServer: true` locally caused Playwright to inherit the dev server's env (which lacked `USE_MOCK_FORM_SCAN=true`). Switching to a dedicated port with `reuseExistingServer: false` ensures the test server always starts fresh with mock mode on, regardless of whether a dev server is running.

**Note for developers:** Next.js prevents two `next dev` instances in the same project directory simultaneously. Stop the dev server before running `npx playwright test`, or use a separate terminal. The test server auto-starts and auto-stops on its own port (3001).

## Task 2: E2E test fixes

**Changed:** `pool-app/e2e/import-from-paper.spec.ts`

Two selector/assertion fixes were needed:

**Fix 1 — Strict mode violation:**
The wait condition `.or(getByText(/dev mode/i))` caused a 2-element strict-mode failure because in mock mode both "2 answers extracted" and "Dev mode — sample data..." are visible simultaneously. Removed the redundant `/dev mode/i` branch.

**Fix 2 — Hydration timing on reload:**
The persistence check used `.all()` + `.map(inputValue())` without retry. RHF populates uncontrolled inputs via ref callbacks after client hydration — there's a window where the DOM inputs are empty. Changed to `expect(async () => ...).toPass({ timeout: 8000 })` which retries until the assertion holds.
