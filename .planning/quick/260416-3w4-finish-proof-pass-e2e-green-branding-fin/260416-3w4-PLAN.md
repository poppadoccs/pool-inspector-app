<task type="auto">
  <name>Task 1: Isolate Playwright to port 3001 with fresh server</name>
  <files>pool-app/playwright.config.ts</files>
  <action>
Replace the current webServer block with a port-3001 isolated configuration.

The current config uses port 3000 and `reuseExistingServer: !process.env.CI`, which means
a dev server already running on 3000 (without USE_MOCK_FORM_SCAN=true) gets reused —
causing the import-from-paper test to hit real Gemini and fail on a 1x1 JPEG.

Apply this exact replacement to `pool-app/playwright.config.ts`:

```ts
import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  testDir: "./e2e",
  fullyParallel: false,
  forbidOnly: !!process.env.CI,
  retries: 0,
  workers: 1,
  reporter: "list",
  use: {
    baseURL: "http://localhost:3001",
    trace: "on-first-retry",
  },
  projects: [
    {
      name: "chromium",
      use: { ...devices["Desktop Chrome"] },
    },
  ],
  webServer: {
    command: "next dev -p 3001",
    url: "http://localhost:3001",
    reuseExistingServer: false,   // always fresh — never conflicts with dev on 3000
    timeout: 120000,
    env: {
      USE_MOCK_FORM_SCAN: "true",
    },
  },
});
```

Note: `command` changes from `npm run dev` to `next dev -p 3001` so Playwright
controls the port directly rather than relying on the npm script default.
The `timeout: 120000` (2 min) gives Next.js warm-up time on first run.
